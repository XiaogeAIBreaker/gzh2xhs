# 代码审查检查清单

- 架构与分层
  - 控制器→用例→仓储/提供者→共享库边界清晰
  - 依赖方向正确，无循环依赖
- 设计原则
  - SOLID/DRY 落地，函数职责单一
  - 圈复杂度≤10（ESLint complexity 规则）
- 错误与异常
  - 输入校验、限流、缓存、外部调用错误处理完善
  - 使用统一错误消息与 HTTP 返回
- 指标与监控
  - KPI 埋点齐全（成功/失败/耗时/命中/限流）
  - A/B 分配与变体传递正确
- 性能与资源
  - 无无效等待；并发控制合理；内存拷贝最小化
- 测试与质量
  - 单测覆盖≥90% 行；关键分支/函数达到阈值
  - CI 通过（lint/typecheck/build/test/coverage/sonar）
- 安全与配置
  - 不泄露密钥；日志脱敏；CORS/CSRF 校验
  - 配置通过 `appConfig` 统一管理
