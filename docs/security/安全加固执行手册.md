# 安全加固执行手册

## 边界与入口

- 接入 WAF 与 Anti-DDoS；API Gateway 强制鉴权与速率限制。
- TLS1.2+，关闭弱算法；HSTS 与安全头（CSP、X-Frame-Options）。

## 数据与密钥

- 敏感字段 AES-256/GCM 加密（服务端），密钥托管 KMS。
- 数据库存储开启 TDE；备份加密存储于 COS（WORM）。
- 凭据最小化与短期凭据（STS）；密钥轮换策略与审计。

## 服务间安全

- Service Mesh mTLS；服务白名单与 RBAC；
- 机密管理：TKE Secrets；禁止将密钥写入仓库或镜像。

## 审计与监控

- 操作审计统一入 CLS；保留 1 年以上；
- 安全告警联动（邮件/IM/工单）；误报/漏报复盘。

## 渗透与合规

- 定期 SAST/DAST 与渗透测试；修复跟踪与验证；
- 合规：数据最小化与隐私政策；第三方依赖审计。
